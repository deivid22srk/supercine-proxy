Como usar

1) Subir o proxy

go run ./cmd/server
# ou
go build -o supercine-proxy ./cmd/server && ./supercine-proxy

Acesse http://localhost:8080 para este dashboard. A API está em http://localhost:8080/v1/.

2) Endpoints do proxy

MétodoRotaDescrição
GET/v1/healthHealth-check do proxy.
GET/v1/statsEstatísticas agregadas.
GET/v1/logs?limit=100Últimas N entradas de log.
POST/v1/logs/clearLimpa logs e stats.
GET/v1/api/<type>Proxy direto para /wp-json/api/<type>.
GET/v1/auth/plansWrapper dos planos de assinatura.
GET/v1/embed?imdb=...&type=...Lista os <server-selector> da página embed.
GET/v1/extract?imdb=...&type=...&server=0Resolve um IMDB para URL direta mp4/m3u8.
GET/v1/extractorsLista hosters suportados.
GET/v1/routesDescoberta automática das rotas upstream.

3) Curl de exemplo

curl 'http://localhost:8080/v1/extract?imdb=tt2250912&type=movies'

4) Variáveis de ambiente

  • LISTEN_ADDR — endereço de bind (default :8080)
  • UPSTREAM_BASE — base do upstream (default https://supercine-tv.net/wp-json)
  • EMBED_BASE — base do embed (default https://supercine-tv.net/embed-api/)
  • CACHE_TTL — TTL do cache (default 5m)
  • CACHE_MAX_ENTRIES — máximo de entradas no cache (default 1000)
  • LOG_MAX_ENTRIES — máximo de logs em memória (default 500)
  • REQUEST_TIMEOUT — timeout HTTP (default 20s)
  • ADMIN_TOKEN — token para endpoints mutantes (default vazio)
  • VERBOSE — log detalhado (default false)
